Escapebar

The most popular Room Escape Games online database in Taiwan. User can view basic information and reviews of games, then post personal reviews and get achievements through contributing contents.

Visit Website

Project Overview

Currently, Escapebar mainly represents as a web appilication which includes one frontend server and three backend service behind.
When user enter Escapebar through browser, the frontend server will request necessary product data from backend for frontend (BFF) server and response server-side rendered document by Next.js.
React takes the rest rendering works triggered by client. Browser will request necessary user data if the user has been authenticated.
To improve performance, use browser cache and service worker cache to prevent unnecessary data requests and render faster. BFF also cache product data in memory for rapid response.
(Under development) User can send request through LINE chatbot, the sentence will be parsed by Dialogflow and return the intent to BFF, and BFF responses possible result. User can interact with the result and open web page in LIFF(LINE Front-end Framework) with Vue.js framework.

Main Features

1. Adopted Server-Side Rendering (SSR) and Incremental Static Regeneration (ISR) for faster rendering.
2. Provides meta data and rich results for better search engine results.
3. Adopted Progressive Web Application (PWA) for service workder cache and resulted in higher retention rate.
4. Adopted code splitting and lazy loading for better web vitals performance.
5. Tracks user behavior properly for understanding and improving User Experience.

Frontend Design and Performance

For better rendering performance, adopted both Server-Side Rendering (SSR) and Incremental Static Regeneration (ISR) powered by Next.js. SSR for dynamic content with high updating frequency, ISR for dynamic content with low updating frequency.

Routing: Next Router.

Code splitting and lazy loading: Next Dynamic Import.

State management: Redux, Context API.

UI libraries: Material-UI, Ant Design.

File caching: Next PWA.

1. PageSpeed Insight performance score raised from 10 to 70 due to SSR, code splitting and faster API response.
2. By feeding Google Search Engine rich results, most product pages could rank in TOP 10 when specific product keywords be searched, and ISR pages could rank in TOP 5 in similar topics.
3. Improved user retention rate 450% by supporting Progressive Web Application.

Backend System Design and Performance

Currently, Escapebar backend system includes three services.

Content Management System: WordPress as headless CMS, provide data to BFF and frontend through RESTful API. Runs on GCP Compute Enegine VM.

Serverless Backend Service: Firebase for authentication, database(NoSQL), file storage functions. Mainly stores user data and user generated content.

Backend For Frontend: Built by Node.js and Express.js, provides RESTful API to handle requests from frontend and does cron jobs those should be done periodically. This server runs as container for the advantage of auto scaling on GCP Cloud Run.

In the earlier version, frontend requests Firebase data through Firebase Cloud Functions, but the responding time is long due to its "cold start" feature.
For fast reponse, move the business logic from Cloud Functions to Cloud Run, and the repsonding time reduced to 10% (from 3,000ms to 300ms) after this change.
When frontend requests product data which are low updating frequency, the BFF server provide data from memory cache, this feature reduced responding time to 20% (300ms to around 60ms).
